Sistema de Gestão Hospitalar

Caio Lopes e Luisa Longo

Funcionamento do Projeto

Nosso sistema é uma plataforma médica de gestão desenvolvida em C++ com uma interface funcional projetada no Qt Creator. O sistema foi arquitetado seguindo os princípios da programação orientada a objetos, garantindo uma estrutura organizada, escalável e de fácil manutenção.

Principais Funcionalidades

O sistema oferece autenticação segura para usuários, garantindo acesso diferenciado para administradores, médicos e pacientes. A gestão de atendimentos permite visualização, agendamento e cancelamento de consultas e cirurgias de forma eficiente. Além disso, possibilita o gerenciamento de cadastros e controle de materiais médicos, bloqueando agendamentos caso os insumos estejam insuficientes.

Arquitetura do Sistema

O sistema é baseado em uma estrutura modular composta por 13 classes distintas, garantindo organização e escalabilidade. Cada classe possui um arquivo header e um arquivo de interface, sendo executadas pelo main.cpp. A autenticação é gerida pela classe Login, que conecta-se ao banco de dados para validar credenciais. Interfaces como ADM_inicio, Medico_inicio e Paciente_inicio direcionam os usuários às funcionalidades adequadas. Além disso, módulos específicos permitem o gerenciamento de atendimentos, cadastros e controle de materiais, assegurando o funcionamento eficiente do sistema.

Banco de Dados

O sistema utiliza um banco de dados SQLite para armazenar e gerenciar informações essenciais.


Ele é estruturado em quatro tabelas principais:


  • Médicos: Armazena ID, nome, username, senha e especialidade.
  • Pacientes: Contém ID, nome, username e senha.
  • Atendimentos: Inclui ID, médico_id, paciente_id, data, horário e tipo de atendimento.
  • Materiais: Controla ID, nome e quantidade de insumos médicos disponíveis.

As tabelas possuem relacionamentos entre si para garantir a integridade dos dados. Por exemplo, a tabela de atendimentos utiliza IDs de médicos e pacientes para associar consultas e procedimentos a cada usuário.
